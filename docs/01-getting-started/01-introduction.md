# 1. Claude Code 소개

> Claude Code는 Anthropic이 만든 터미널 기반 AI 에이전트 코딩 도구입니다. 코드베이스를 읽고, 파일을 편집하고, 명령어를 실행하며, 개발 도구와 통합됩니다.

---

## Claude Code란?

Claude Code는 **터미널에서 동작하는 자율형(agentic) AI 코딩 어시스턴트**입니다. 일반적인 코드 자동완성 플러그인과 달리, Claude Code는 여러분의 전체 코드베이스를 이해하고, 계획을 세우고, 여러 파일에 걸쳐 변경을 수행하고, 결과를 스스로 검증하는 **에이전트**입니다.

```
사용자: "인증 버그를 고쳐줘"

Claude Code:
  1. 관련 파일을 검색하고 읽음
  2. 여러 파일의 맥락을 파악
  3. 조율된 수정을 여러 파일에 걸쳐 수행
  4. 테스트를 실행하여 수정을 검증
  5. 요청 시 커밋까지 생성
```

Claude Code는 터미널, VS Code, JetBrains IDE, 브라우저, 데스크톱 앱 등 다양한 환경에서 사용할 수 있으며, 어디서든 동일한 엔진으로 동작합니다.

---

## 에이전틱 루프 (Agentic Loop)

Claude Code의 핵심 아키텍처는 **세 단계의 반복 루프**입니다:

### 1단계: 컨텍스트 수집 (Gather Context)

- 코드베이스를 읽어 관련 코드를 이해
- 파일명 패턴이나 내용으로 파일을 검색
- 프로젝트 구조와 의존성 탐색
- 에러 메시지와 명령어 출력 분석
- CLAUDE.md에서 프로젝트별 지침 로드

### 2단계: 행동 (Take Action)

- 여러 파일에 걸쳐 조율된 편집 수행
- 셸 명령어, 테스트, 빌드 실행
- Git을 이용한 버전 관리
- MCP를 통해 외부 서비스와 상호작용
- 서브에이전트를 생성하여 병렬 작업 수행

### 3단계: 검증 (Verify Results)

- 테스트를 실행하여 수정이 동작하는지 확인
- 컴파일과 린팅 검사
- 기대 결과와 실제 출력 비교
- 필요시 사용자에게 피드백 요청

```
┌─────────────────────────────────────────────┐
│                                             │
│    ┌──────────┐   ┌──────────┐   ┌──────┐  │
│    │ 컨텍스트 │──▶│   행동   │──▶│ 검증 │  │
│    │   수집   │   │          │   │      │  │
│    └──────────┘   └──────────┘   └──┬───┘  │
│         ▲                           │      │
│         └───────────────────────────┘      │
│              (필요시 반복)                   │
└─────────────────────────────────────────────┘
```

이 세 단계는 선형적이지 않습니다. Claude는 하나의 작업 내에서 여러 번 이 루프를 반복하며, 검증 결과를 바탕으로 새로운 컨텍스트를 수집하고 접근 방식을 조정합니다. 그리고 사용자는 언제든 이 루프에 개입하여 방향을 수정할 수 있습니다.

## 지원 모델

Claude Code는 세 가지 모델을 지원하며, 상황에 따라 전환할 수 있습니다.

### Claude Opus 4.6 — 최강의 추론 능력

| 항목 | 내용 |
|------|------|
| **특징** | 가장 강력한 추론, 에이전트 구축 및 코딩에 최적화 |
| **컨텍스트 윈도우** | 200K (기본) / 1M (베타, `opus[1m]`) |
| **최대 출력** | 128K 토큰 |
| **적응형 추론** | 작업 복잡도에 따라 사고 깊이를 자동 조절 |
| **노력 수준** | low / medium / high 선택 가능 |
| **API 가격** | $5 / 1M 입력 토큰, $25 / 1M 출력 토큰 |
| **적합한 용도** | 복잡한 아키텍처 결정, 멀티스텝 추론, 대규모 리팩토링 |

### Claude Sonnet 4.6 — 균형잡힌 추천 모델

| 항목 | 내용 |
|------|------|
| **특징** | 속도와 지능의 최적 균형, 대부분의 작업에 추천 |
| **컨텍스트 윈도우** | 200K (기본) / 1M (베타, `sonnet[1m]`) |
| **최대 출력** | 64K 토큰 |
| **API 가격** | $3 / 1M 입력 토큰, $15 / 1M 출력 토큰 |
| **적합한 용도** | 일상적 코딩, 기능 구현, 버그 수정, 코드 리뷰 |

### Claude Haiku 4.5 — 빠르고 효율적

| 항목 | 내용 |
|------|------|
| **특징** | 가장 빠른 추론 속도, 프론티어에 근접한 지능 |
| **컨텍스트 윈도우** | 200K |
| **최대 출력** | 64K 토큰 |
| **API 가격** | $1 / 1M 입력 토큰, $5 / 1M 출력 토큰 |
| **적합한 용도** | 간단한 작업, 빠른 반복, 서브에이전트 백그라운드 처리 |

모델 전환은 세션 중 `/model` 명령어로, 또는 시작 시 `--model` 플래그로 할 수 있습니다:

```bash
# 세션 중 모델 전환
/model opus
/model sonnet
/model haiku

# 시작 시 모델 지정
claude --model opus
```

> **팁**: `opusplan` 별칭을 사용하면 계획 수립 시에는 Opus, 코드 실행 시에는 Sonnet을 자동으로 전환하여 비용과 품질의 균형을 맞출 수 있습니다.

## 다른 AI 코딩 도구와의 차이점

### 아키텍처 차이

| 도구 | 유형 | 작동 방식 |
|------|------|-----------|
| **GitHub Copilot** | 인라인 자동완성 | IDE 안에서 타이핑할 때 코드 제안 |
| **Cursor** | IDE 통합 에이전트 | VS Code 기반 에디터 안에서 동작 |
| **Claude Code** | 터미널 자율 에이전트 | 독립적으로 터미널에서 실행, 전체 코드베이스 대상 |

### 핵심 차별점

**1. 코드베이스 규모 처리**
Claude Code는 최대 200K 토큰(1M 베타)을 처리할 수 있어, 대규모 코드베이스의 심층 아키텍처 리뷰와 전략적 결정에 적합합니다.

**2. 토큰 효율성**
Anthropic의 [엔지니어링 블로그](https://www.anthropic.com/engineering/claude-code-best-practices)에 따르면, Claude Code는 동일한 작업에 대해 경쟁 도구 대비 **5.5배 적은 토큰**을 사용했습니다.

**3. 자율성**
Copilot은 제안을 수락/거절하는 방식이고, Cursor는 IDE 안에서 개발자가 AI를 지시하는 방식입니다. Claude Code는 작업을 설명하면 스스로 계획하고 실행하는 **위임(delegation)** 방식입니다.

**4. 통합 범위**
Claude Code는 터미널, VS Code, JetBrains, 웹, 데스크톱, Slack, CI/CD 등 어디서든 동일한 엔진으로 동작합니다.

> **한 줄 요약**: Copilot은 일상적 코딩을 빠르게, Cursor는 대규모 프로젝트를 관리하기 좋게, Claude Code는 복잡한 문제를 이해하기 좋게 만들어줍니다. 최고의 개발자들은 하나의 도구에 의존하지 않고, 상황에 맞는 도구를 선택합니다.

## 주요 사용 사례

### 반복 작업 자동화
- 테스트되지 않은 코드의 테스트 작성
- 프로젝트 전체의 린트 에러 수정
- 머지 충돌 해결
- 의존성 업데이트와 릴리스 노트 작성

### 기능 구현과 버그 수정
- 자연어로 원하는 기능 설명 → Claude가 계획 수립, 코드 작성, 검증
- 코드베이스를 추적하여 버그의 근본 원인 파악

### Git 및 PR 워크플로우
- 변경 사항 스테이징과 커밋 메시지 자동 생성
- 브랜치 생성과 PR 오픈
- GitHub Actions / GitLab CI와 통합하여 자동 리뷰 및 트리아지

### 외부 도구 연결 (MCP)
- Google Drive에서 디자인 문서 읽기
- Jira 티켓 업데이트
- Slack에서 데이터 가져오기
- 커스텀 도구 연동

### 멀티 에이전트 협업
- 여러 Claude Code 에이전트를 서로 다른 작업에 투입
- 리드 에이전트가 작업을 조율하고 결과를 병합
- 에이전트 팀이 병렬로 작업

### Unix 스타일 자동화
- 로그를 Claude Code에 파이프로 전달
- CI/CD 파이프라인에서 실행
- 다른 Unix 도구와 체이닝

## 접근 방법과 요금

Claude Code는 **무료로 설치**할 수 있으며, 사용하려면 다음 중 하나가 필요합니다:

### 구독 플랜

| 플랜 | 가격 | 포함 내용 |
|------|------|-----------|
| **Claude Pro** | $20/월 | Sonnet 4.6, 5배 사용량, Extended Thinking |
| **Claude Max 5x** | $100/월 | Sonnet 4.6 5배 + Opus 4.6 접근, 1M 컨텍스트 |
| **Claude Max 20x** | $200/월 | Sonnet 4.6 20배 + Opus 4.6 최대 사용량 |
| **Team** | $25~30/인/월 | 팀 협업, 조직 관리 |

### API (종량제)

| 모델 | 입력 | 출력 |
|------|------|------|
| Opus 4.6 | $5 / 1M 토큰 | $25 / 1M 토큰 |
| Sonnet 4.6 | $3 / 1M 토큰 | $15 / 1M 토큰 |
| Haiku 4.5 | $1 / 1M 토큰 | $5 / 1M 토큰 |

> **참고**: 평균 비용은 개발자당 하루 약 $6이며, 90%의 사용자가 하루 $12 이하를 사용합니다.

### 서드파티 프로바이더
- **Amazon Bedrock**: AWS 자격 증명과 빌링 사용
- **Google Vertex AI**: GCP 자격 증명과 빌링 사용
- **Microsoft Foundry**: Azure 자격 증명과 빌링 사용

## 이 책에서 배울 것들

이 책은 Claude Code의 모든 기능을 초급부터 전문가 수준까지 다룹니다:

- **Part I** (1~4장): 설치부터 기본 사용법까지
- **Part II** (5~8장): 슬래시 커맨드, 도구, 모델 선택 등 핵심 기능
- **Part III** (9~13장): CLAUDE.md, 권한, Rules, 메모리 등 설정 심화
- **Part IV** (14~18장): Git, Plan 모드, 세션 관리 등 실전 워크플로우
- **Part V** (19~24장): Skills, Hooks, MCP, 서브에이전트 등 고급 기능
- **Part VI** (25~28장): IDE 통합, CI/CD, 클라우드 프로바이더, 엔터프라이즈
- **Part VII** (29~32장): 실전 레시피, 비용 최적화, 보안
