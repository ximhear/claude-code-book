# 3. 첫 번째 세션

> Claude Code를 처음 실행하고, 파일을 읽고, 편집하고, 명령어를 실행하는 기본적인 상호작용을 체험합니다.

---

## Claude Code 시작하기

프로젝트 디렉토리에서 `claude`를 실행합니다:

```bash
cd ~/my-project
claude
```

Claude Code가 시작되면 인터랙티브 프롬프트가 나타납니다:

```
╭────────────────────────────────────────╮
│ Claude Code                     v2.1.x │
│                                        │
│ /help for commands                     │
╰────────────────────────────────────────╯

>
```

이 시점에서 Claude는 다음에 접근할 수 있습니다:
- **프로젝트 파일**: 현재 디렉토리와 하위 디렉토리의 모든 파일
- **터미널**: 여러분이 실행할 수 있는 모든 명령어 (빌드 도구, git, 패키지 관리자 등)
- **Git 상태**: 현재 브랜치, 커밋되지 않은 변경 사항, 최근 커밋 히스토리
- **CLAUDE.md**: 프로젝트별 지침 파일 (있는 경우)

---

## 첫 대화

자연어로 원하는 것을 말하면 됩니다:

```
> 이 프로젝트의 구조를 설명해줘
```

Claude는 프로젝트를 탐색하고 구조를 분석하여 응답합니다. 이 과정에서 Claude는 여러 도구를 사용합니다:

1. **Glob** — 파일 패턴으로 파일 목록 탐색
2. **Read** — 파일 내용 읽기
3. **Grep** — 파일 내용에서 패턴 검색

이 모든 것은 읽기 전용 작업이므로 별도의 승인 없이 자동으로 실행됩니다.

---

## 권한 승인 이해하기

Claude가 시스템을 변경하는 작업을 하려면 사용자의 **승인**이 필요합니다.

### 승인이 필요 없는 작업 (읽기 전용)
- 파일 읽기 (Read)
- 파일 검색 (Glob, Grep)
- 디렉토리 목록 조회

### 승인이 필요한 작업
- **파일 편집** (Edit, Write): 기존 파일 수정이나 새 파일 생성
- **명령어 실행** (Bash): 셸 명령어 실행
- **웹 요청** (WebFetch): 외부 URL 접근

Claude가 승인이 필요한 작업을 시도하면 다음과 같은 프롬프트가 나타납니다:

```
Claude wants to run: npm test

  Allow?
  [y] Yes  [n] No  [a] Always allow this  [d] Don't ask again for this session
```

선택 옵션:
- **y (Yes)**: 이번 한 번만 허용
- **n (No)**: 거부
- **a (Always allow)**: 이 명령어를 항상 허용 (설정에 저장)
- **d (Don't ask)**: 이 세션 동안 이 명령어를 묻지 않음

### 파일 편집 승인

파일 편집 시에는 변경 내용의 diff가 표시됩니다:

```
Claude wants to edit: src/utils.ts

  - function validate(input) {
  + function validate(input: string): boolean {
      if (!input) return false;
  -   return true;
  +   return input.length > 0;
    }

  Allow?
  [y] Yes  [n] No  [a] Always allow edits
```

## 기본 작업 체험

### 파일 읽기

```
> src/index.ts 파일을 보여줘
```

Claude는 `Read` 도구로 파일을 읽어 내용을 보여줍니다.

### 파일 편집

```
> src/utils.ts의 validate 함수에 타입 어노테이션을 추가해줘
```

Claude는 파일을 읽고, 변경 계획을 세우고, `Edit` 도구로 수정합니다. 수정 전에 diff를 보여주며 승인을 요청합니다.

### 명령어 실행

```
> 테스트를 실행해줘
```

Claude는 프로젝트의 테스트 러너를 파악하고 (예: `npm test`, `pytest`, `go test`) 적절한 명령어를 실행합니다. 실행 전에 승인을 요청합니다.

### 새 파일 생성

```
> src/helpers/ 디렉토리에 string-utils.ts 파일을 만들어줘.
> 문자열 유틸리티 함수들을 넣어줘.
```

Claude는 `Write` 도구로 새 파일을 생성합니다.

### 여러 파일에 걸친 변경

```
> API 엔드포인트에 입력 검증을 추가해줘.
> 관련 코드는 src/routes/에 있어.
```

Claude는 여러 파일을 읽고, 변경이 필요한 곳을 파악하고, 조율된 수정을 수행합니다.

---

## 대화의 흐름

Claude Code는 **대화형**입니다. 완벽한 프롬프트가 필요하지 않습니다. 시작하고, 다듬어가면 됩니다:

```
> 로그인 버그를 고쳐줘

[Claude가 조사하고 수정을 시도]

> 그건 아닌 것 같아. 문제는 세션 핸들링 쪽이야.

[Claude가 접근 방식을 수정]

> 이번엔 맞는 것 같아. 테스트도 실행해줘.

[Claude가 테스트 실행하여 검증]
```

첫 시도가 맞지 않아도 처음부터 다시 시작할 필요가 없습니다. **반복하며 다듬어가면** 됩니다.

### 작업 중간에 끼어들기

Claude가 잘못된 방향으로 가고 있다면, 타이핑을 시작하고 Enter를 누르면 됩니다. Claude는 하던 작업을 멈추고 여러분의 입력을 기반으로 접근 방식을 조정합니다.

---

## 세션 종료와 비용 확인

### 비용 확인

세션 중 `/cost` 명령어로 현재까지의 토큰 사용량과 비용을 확인합니다:

```
> /cost

Total cost:            $0.55
Total duration (API):  6m 19.7s
Total duration (wall): 6h 33m 10.2s
Total code changes:    42 lines added, 12 lines removed
```

| 항목 | 설명 |
|------|------|
| **Total cost** | API 토큰 비용 (구독자에게는 참고용) |
| **Total duration (API)** | API 호출에 소요된 실제 시간 |
| **Total duration (wall)** | 세션 시작부터 현재까지의 경과 시간 |
| **Total code changes** | 추가/삭제된 코드 라인 수 |

> **참고**: `/cost`는 API 사용자를 위한 것입니다. Claude Pro/Max 구독자는 사용량이 구독에 포함되므로 `/stats`로 사용 패턴을 확인하세요.

### 세션 종료

세션을 종료하려면:

- `/clear` — 대화를 지우고 새로 시작 (세션 유지)
- `Ctrl+C` — 현재 작업 중단
- `Ctrl+D` 또는 `exit` — Claude Code 종료

세션은 **자동으로 로컬에 저장**됩니다. 다음에 같은 디렉토리에서 `claude --continue`로 이어서 작업할 수 있습니다.

---

## 안전장치: 체크포인트

Claude가 파일을 편집하기 전에 **자동으로 현재 상태를 스냅샷**합니다. 문제가 생기면:

- **Esc 두 번**: 이전 체크포인트로 되돌리기
- **Claude에게 요청**: "방금 변경을 되돌려줘"

체크포인트는 세션에 로컬이며, git과는 별도입니다. 파일 변경만 커버하며, 원격 시스템에 영향을 미치는 작업(DB, API, 배포)은 체크포인트 대상이 아닙니다.

---

## 첫 세션 실습 예제

다음 순서로 Claude Code를 체험해보세요:

```
# 1. 프로젝트 디렉토리에서 시작
cd ~/my-project
claude

# 2. 프로젝트 구조 파악
> 이 프로젝트의 구조와 주요 파일을 설명해줘

# 3. 특정 파일 분석
> src/app.ts를 읽고 주요 로직을 설명해줘

# 4. 간단한 수정
> README.md의 설치 방법 섹션을 업데이트해줘

# 5. 테스트 실행
> 테스트를 실행하고 결과를 알려줘

# 6. 비용 확인
> /cost

# 7. 종료
> exit
```
